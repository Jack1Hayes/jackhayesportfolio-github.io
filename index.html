Recipe Suggestor

This project uses both a range of different packages and skills to develop a recipe suggester based on a dataset of users and recipes from food.com to create a GUI that allows the user to find a recipe based on ingredients input into the program.

To begin I used the data from food.com to show recipe rating distribution#group recipies and user IDs then I did a short analysis for users and recipies, mapped the values of rating into the train and test data sets and applied centered cosine to the dataframe to minimize the differences between the “hard raters” and “easy raters.” To measure the performance of this recommendation engine in the next steps, I split the dataframe into ¾ train data set and ¼ test data set. There are two major types of memory-based collaborative filtering. User-based: “Users who are similar to you also like …” Item-based: “Users who like this item also like …”, For this we will use user based to make the GUI simpler.

Once the model was created I used TKinter to generate a user Frienly GUI. This GUI is able to be reused and changed upon the user command and takes images from the food.com website using beautiful soup for each recipe the user picks to give the user a clearer idea. To improve I would like to have the model seperatly contained so as not to load it everytime. Get immages also in a seperate database to decrease loading times, include images for suggested recipies and also improve the UI/UX design.

Twitter Disaster Preidictor

For this project I used a dataset from kaggle.com, that takes tweets of different incidents and tells us if those tweets are real of fake. From this information I created a model taht analysis the language of a disaster tweet and measures whether or not it is real or not.

First was to clean the data, removing any unecessary words using the ntlk stop words before tokenizing and vecotizing using sklearn packages the words to use later in the model.

Once the data was cleaned I split the data into a test and train set before using tensorflow to create a model that lookeed at the numbers from the ntlk vectorization and predicted the liklyhood of each tweet being real between 0, 1.

Finally I analysed the model, looking at the test data and seeing the probability density of each prediction. Also using a confusion matrix I got an accuracy of 0.78
